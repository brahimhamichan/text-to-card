# Contributing

Thanks for helping improve txt2card.

## Workflow

1. Fork the repository
2. Create a branch for your change
3. Make focused changes — keep the code simple (YAGNI)
4. Run the full suite:

```sh
npm run test:all
```

5. Open a pull request with a short description of what and why

## Code style

- Prefer simple, readable code over abstractions
- Avoid new dependencies unless necessary
- Match existing patterns in `cli.js` and tests
- Do not expand scope beyond the change at hand

## Questions

Open an issue on [GitHub](https://github.com/brahimhamichan/text-to-card) if you need clarification before starting larger work.
